Detailed Description

Kernel calls can be structured such that Kernel_t::kernel is either:

  • A non-factorizable function, or
  • A thin wrapper around Kernel_t::kernel_unit_unified with a guard like if (i<n) kernel_unit_unified(i, n, args...);.

This dispatcher does not validate memory-placement compatibility of the arguments. If argument memory locations are arranged properly and kernel_unit_unified calls are factorizable, the run_kernel struct can execute the kernel in parallel (GPU) or in loop/OpenMP fallback (CPU), depending on runtime GPU usability.

Typical usage:

  • Define a kernel type with static kernel(...), or with kernel_unit_unified(...) and a guarded kernel(...) wrapper.
  • Invoke via one of:
    • run_kernel<example_kernel>(shared_mem_size, stream).parallel_1D(n, args...);
    • run_kernel<example_kernel>(shared_mem_size, stream).parallel_2D(nx, ny, args...);
    • run_kernel<example_kernel>(shared_mem_size, stream).parallel_3D(nx, ny, nz, args...);
